Setting Up Your Project Environment

Let's get started with setting up your project structure! We'll create a new project folder, set up your HTML file, and learn how to properly link images using relative file paths.

Step 1: Create Project Folder

Navigate to your HTML folder within the developer folder and create a new folder named "Project 2 Invitation". This will be where all your project files live. 


📂 developer
  📂 html
    📂 Project 2 Invitation

Step 2: Create HTML File

Inside your new "Project 2 Invitation" folder, create a new file named index.html. This will be your main HTML file for the project. 


📂 developer
  📂 html
    📂 Project 2 Invitation
        📄 index.html

Step 3: Add HTML Skeleton

Open your index.html file and add the basic HTML structure. This includes the DOCTYPE declaration, html, head, and body tags. 


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Project 2 - Event Invitation</title>
</head>
<body>

</body>
</html>

Step 4: Understanding File Structure

Let's understand how your files are organized. Your directory structure should look like this: 


📂 developer
  📂 assets
    📄 your-image.jpg    
  📂 html
    📂 Project 2 Invitation
        📄 index.html

Step 5: Add Image Using Relative Path

Inside your <body> tag, add an <img> tag that references your image from the assets folder. Since we're in the Project 2 Invitation folder, we'll need to navigate up to reach the assets folder. 


<img src="../../assets/your-image.jpg" alt="Description of your image">
Understanding Relative Paths:
• ../ means "go up one directory level"
• First ../ moves up from "Project 2 Invitation" to "html"
• Second ../ moves up from "html" to "developer"
• Then we can go into the assets folder to reach your image

Step 6: Verify Your Work

Open your index.html file in a browser to make sure:

  • The HTML skeleton is properly formatted
  • The image appears on the page
  • There are no broken image links
Troubleshooting: If your image doesn't appear, double-check your file path. Remember that you need to go up two levels (../../) to get from your project folder to the developer folder, then into assets.
