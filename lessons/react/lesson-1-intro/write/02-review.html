Review: Setting Up a React App with CDNs and Components

Let's review the steps to set up a basic React application using CDNs and organizing multiple components in separate files.

1. HTML Structure

In the index.html file, we include the necessary CDN links for React, ReactDOM, and Babel, and set up <script> tags to load each component and the main entry files. 


<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>React App</title>
  <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
  <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
  <script src="https://unpkg.com/@babel/standalone@7/babel.min.js"></script>
</head>
<body>
  <div id="root"></div>
  <script type="text/babel" src="/components/FoodList.js" defer></script>
  <script type="text/babel" src="/components/Essentials.js" defer></script>
  <script type="text/babel" src="/components/Clothes.js" defer></script>
  <script type="text/babel" src="App.js" defer></script>
  <script type="text/babel" src="main.js" defer></script>
</body>
</html>

2. Main Rendering File (main.js)

In the main.js file, we render our main component App to the root element. 


const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(<App />);
Tip: Using this approach, our main.js file remains clean and focused on rendering the main component. Any additional logic or setup can be added to the App component.

3. Setting up the App.js

In the app.js file, we render any additional components or elements needed for our application. 


          // App.js
          function App() {
            return (
              <div className="App">
                  <h1>Shopping List</h1>
                  <FoodList />
                  <Essentials />
                  <Clothes />
              </div>
            );
          }
Tip: Using this approach, our App.js file's main purpose is to render our other components. This allows us to keep our code organized and maintainable.

3. Creating Components

Each component is stored in its own file. For example, FoodList, Essentials, and Clothes each contain a specific list and are written in a modular way.

Example: FoodList.js

function FoodList(){
  return(
    <div>
