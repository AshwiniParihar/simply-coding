Components in React

Components are the building blocks of React applications. Components allow you to break your project into smaller sections. Each section can manage its own functionality and appearance. They are also reusable, which means you can use the same component in multiple places in your application. This makes your code more organized and easier to maintain.

Basic Structure


    function <ComponentName>() {
    
        const headerText = "Component Name";
    
        // Above the return statement is for JavaScript logic
    
        return (
            // Everything below the return statement is JSX
            <element>
                <p>{headerText}</p>
                <button>Home</button>
                <button>About</button>
            </element>
        );
    }

When put into action, it looks like this


    function Navbar(){

        const headerText = "Navbar";

        // Above the return statement is for JavaScript

        return (
            // Everything below the return statement is JSX
            <nav>
                <p>{headerText}</p>
                <button>Home</button>
                <button>About</button>
            </nav>
        )
    }

Components are built as functions. Otherwise known as functional components. But instead of returning JavaScript code they return our JSX which gets rendered to the DOM. We can do any JavaScript logic we need above the return statement. We can even use the values of those variables in our JSX using curly braces.

CDN Setup for Components

When using React components with a CDN, it's crucial to include a script tag for each component you create. This ensures that the component is properly imported and available for use in your application. 


        <script src="https://cdn.example.com/MyComponent.js"></script>

Be sure to replace https://cdn.example.com/MyComponent.js with the actual URL where your component is hosted. Additionally, it's important to load the script tags in the correct order: your main index.js script should be the last script included. This way, it has access to all your other components and files, allowing your application to function correctly.

You can place each script tag in the <head> section or at the end of the <body> section of your HTML file, depending on your setup.

Common Structure: App.js and index.js

In a typical React application, you'll find an App.js file that acts as the root component. This component is where you can render other components and manage the overall structure of your application. 


    import React from 'react';
    import ReactDOM from 'react-dom';
    import './index.css'; // Import your CSS file
    import App from './App'; // Import your main App component
    
    const root = ReactDOM.createRoot(document.getElementById("root"));
    root.render();

In the index.js file, we use ReactDOM.createRoot to create a root for our React application and then call root.render() to render our App component into the DOM. This structure allows you to maintain a clean separation of components and helps with scalability as your application grows.

Best Practices
  • Name components using PascalCase, like MyComponent, for consistency.
  • Break down complex interfaces into smaller, reusable components to make your code more manageable.
  • Test each component as you build it to ensure it renders correctly before adding more complexity.
  • Give components meaningful names that reflect their purpose, making it easier to understand and maintain. Header, Footer, Navigation, etc.

Using {} in our component.

Using our curly braces {} in our JSX allows us to use JavaScript expressions. Which means we can use variables, functions, and more! This gives us the power to use dynamic content in our JSX.

Example: 

function MyComponent() {
    const name = "John Doe";
    return (
        <div>
            <h1>Hello, {name}!</h1>  
        </div>
    )
}

Reminder: Any JavaScript used after the return statement should be to render to the UI, not for building functions or logic.

Single Element

React Components can only have a single root element. This means that you can only have one top-level element. But that element can contain as many children as needed.

Example: 

function Navbar(){

    const headerText = "Navbar";

    return (
        <nav>
            <p>{headerText}</p>
            <button>Home</button>
            <button>About</button>
        </nav>
    )
}

Our Navbar component above is rendering a single nav element, but the nav element can contain as many children elements as we need.

File Structure

One of the benefits of React is that it is modular, allowing us to break our code up into smaller sections easily. If we put all of our components into a single file we are missing out on one of the key benefits of React. Each component should be in its own file, and the file name should match the name of the component. It is a common practice to have a components folder in our project and put all of our components in there.
