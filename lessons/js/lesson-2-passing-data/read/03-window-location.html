Understanding window.location in JavaScript

What is window.location?

The window.location property in JavaScript allows us to access and manipulate the URL of the current page. We can use it to navigate to a new page, reload the page, or get information about the current URL.

Common Properties of window.location

  • window.location.href: Gets or sets the URL of the current page. Setting it to a new URL redirects the browser to that page.
  • window.location.pathname: Returns the path of the current URL (e.g., "/page.html").
  • window.location.hostname: Returns the domain name of the web host (e.g., "example.com").
  • window.location.protocol: Returns the protocol of the URL (e.g., "http:" or "https:").
  • window.location.reload(): Reloads the current page.

Example: Redirecting After Form Submission

Below is a form that, upon submission, uses window.location.href to redirect the user to a new page. The preventDefault() method is used to stop the form's default behavior so we can add the custom redirect logic.

You'll notice URI in this example, you can review that in detail in the next section!

  
  
  <form id="redirectForm">
    <label for="name">Enter your name:</label>
    <input type="text" id="name" name="name" placeholder="Your Name">
    <button type="submit">Submit</button>
  </form>

  <script>
    document.getElementById('redirectForm').addEventListener('submit', function(event) {
      event.preventDefault(); // Stop the default form submission

      // Capture the input
      let userName = document.getElementById('name').value;

      // Redirect to another page, including the user's name in the URL query string
      window.location.href = 'thankyou.html?name=' + encodeURIComponent(userName);
    });
  </script>

Explanation of the Code

In the example above:

  • We listen for the form's submit event and use preventDefault() to stop the default submission behavior.
  • The user's name is captured from the input field and stored in userName.
  • window.location.href is then set to a new URL (in this case, "thankyou.html") with the user's name added as a query parameter.
  • The encodeURIComponent() function is used to ensure any special characters in the user input are safely added to the URL.
  • The URI will be reviewed in the next section.
