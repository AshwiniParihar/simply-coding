Introduction to CSS Specificity

Now that you’ve learned how to apply basic CSS selectors, it’s time to look ahead and understand how these rules are prioritized.

What is Specificity?
Specificity determines which CSS styles are applied when multiple selectors target the same element.

How Specificity Works
Each type of selector has its own "weight" or specificity score, which contributes to the overall specificity of a rule. The browser uses this score to determine which style should be applied. Specificity is calculated based on a four-part value represented as (a, b, c, d), where:

a(inline styles)

Inline styles directly written within the HTML element using the style attribute have the highest specificity.

...
b(ID selectors)

Selectors using IDs are more specific than class selectors or element selectors.

...
c(Class selectors, attribute selectors, and pseudo-classes)

These have a lower specificity than ID selectors but higher than element selectors.

...
d(Element selectors and pseudo-elements)

These have the lowest specificity.

...
a(inline styles)
Example: 
<div style="color: red;"/>
b(ID selectors)
Example: 
#myId {color: red;}
c(Class selectors, attribute selectors, and pseudo-classes)
Example: 
.myClass, [type="text"], :hover
d(Element selectors and pseudo-elements)
Example: 
div, p, ::before.

Specificity Calculation Example

Consider the following CSS rules:


        /* Example 1 */
        div {
          color: blue;
        }
        
        /* Example 2 */
        .myClass {
          color: green;
        }
        
        /* Example 3 */
        #myId {
          color: red;
        }

If an HTML element is targeted by all three rules, the browser will prioritize based on specificity:

Html File:


        <div id="myId" class="myClass">Hello, world!</div>

Explanation:

In this example:
  • Example 1 (div) has a specificity of (0, 0, 0, 1).
  • Example 2(.myClass) has a specificity of (0, 0, 1, 0).
  • Example 3(#myId) has a specificity of (0, 1, 0, 0)

Result:The browser applies the rule from Example 3 (#myId) because it has the highest specificity (0, 1, 0, 0).
