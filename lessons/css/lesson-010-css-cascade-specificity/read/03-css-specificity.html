CSS Specificity

Specificity refers to how detailed a CSS selector is in targeting an element. More specific selectors take precedence over less specific ones. The order of specificity is:

  • ID selectors (e.g., #main-heading) have the highest specificity.
  • Class selectors (e.g., .heading) come next.
  • Attribute selectors (e.g., [type="text"]) follow.
  • Element selectors (e.g., h1, p) have the lowest specificity.

#main-heading {
  color: green;
}

.heading {
  color: blue;
}

h1 {
  color: red;
}

In this example, the color will be green because the ID selector #main-heading is the most specific.

Type of CSS

There are three types of CSS that can apply styles to an HTML document:

  • External Stylesheet: Linked to the HTML file through a <link> tag.
  • Internal Stylesheet: Placed inside the <style> tag in the <head> section.
  • Inline Style: Applied directly to an element using the style attribute.

Inline styles override both external and internal styles.

CSS !important Keyword

The !important keyword is used to give a CSS rule higher priority. A rule with !important will override any other rule, regardless of position or specificity. 


h1 {
  color: red !important; /* This will override all other color rules for h1 */
}

h1 {
  color: blue; /* This rule is ignored because the first rule has !important */
}

Categories of Importance

CSS rules can be categorized into four broad categories that affect the cascade:

  • Position: The location of the rule in the CSS file.
  • Specificity: How specific the selector is.
  • Type: The type of CSS rule (External, Internal, or Inline).
  • Importance: The !important keyword.

Position in CSS

Rules that are placed later in the CSS file will override rules placed earlier, provided they have the same specificity. This means that a rule at the bottom of the CSS file can override a rule at the top, even if the rules target the same element. 


/* First rule */
h1 {
  color: red;
}

/* Second rule */
h1 {
  color: blue; /* This rule will be applied because it comes last */
}
