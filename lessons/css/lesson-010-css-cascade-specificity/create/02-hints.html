HTML Structure and CSS for Box Elements

Let's review how to style `div` elements using internal and external CSS. Below, we cover the structure and styling applied to two `div` elements: one with a class and the other with an ID.

1. HTML Structure of Box Elements

In the following structure, we have two `div` elements, each containing a paragraph (`<p>`) with specific styles:

  • <div class="box1">: A `div` element with the class `box1`, which contains a paragraph (`<p>`) styled with inline CSS. The paragraph inside `box1` has its text color set to red (color: red;).
  • <div id="box2">: A `div` element with the `id` of `box2`, which contains another paragraph (`<p>`). The paragraph inside `box2` is not styled inline but is targeted by internal CSS to have green text color (color: green;).
Internal & External CSS for Box Elements
  • Internal CSS for #box2 p: The `p` tag inside the `div` with id="box2" is styled with a green text color (color: green;).
  • External CSS for .box1 and #box2: The .box1 class applies a lightblue background color, width of 300px, height of 100px, and a margin of 20px. The #box2 ID applies a lightgreen background color, width of 300px, height of 100px, and a margin of 20px.
Do This
  • Use class selectors for reusable styles across multiple elements.
  • Apply ID selectors only for unique elements to prevent overriding issues.
  • Leverage pseudo-classes to enhance user interactions (e.g., :hover for buttons).
  • Use attribute selectors for form inputs or specific attributes when needed.
  • Combine selectors to reduce redundancy (e.g., h1, h2, h3).
Don't Do This
  • Don't overuse ID selectors, as they can create high specificity issues.
  • Don't rely on overly specific selectors (e.g., div > p > span), making them hard to maintain.
  • Don't neglect adding :focus or other pseudo-classes for accessibility.
  • Don't apply global styles using the universal selector * unless necessary; it can impact performance.

2. Best Practices

Follow these best practices when using CSS selectors:

  • Keep selectors simple and readable for better maintainability.
  • Use naming conventions like BEM (Block Element Modifier) to maintain consistent structure.
  • Write DRY (Don't Repeat Yourself) code by grouping similar selectors.
  • Utilize pseudo-elements for specific text and styling requirements (e.g., ::before, ::after).
  • Ensure responsive design by using media queries and avoiding hard-coded sizes.
