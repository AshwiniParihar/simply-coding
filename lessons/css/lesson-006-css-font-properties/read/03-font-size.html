Font-size

The `font-size` property defines the size of the text. It can be specified using different units such as `px`, `em`, `rem`, and `pt`. Each unit has its own use case and effect on responsiveness.

Units for Font Size:

  • Pixels (px): Fixed size that does not scale when the browser is resized. Best used for elements that donâ€™t need to adjust based on screen size.
  • Points (pt): Commonly used in print but less so for web design.
  • Ems (em): Relative to the font size of the parent element. If a parent element has a font size of 16px, 1em equals 16px.
  • Rems (rem): Relative to the font size of the root element (``). More consistent for maintaining responsive designs.

Best Practices
  • Use `rem` for consistent, responsive designs.
  • Specify fallback fonts in `font-family` to ensure a smooth user experience.
  • Use Google Fonts for custom font styling.

Example:

      
    <h2>Understanding `font-size`</h2>
    <p style="font-size: 16px;">
    This text is set to 16px.
    </p>
    <p style="font-size: 1em;">
    This text is set to 1em (relative to its parent).
    </p>
    <p style="font-size: 1.5rem;">
    This text is set to 1.5rem (relative to the root element).
    </p>
    <p style="font-size: 12pt;">
    This text is set to 12pt (commonly used in print).
    </p>

OUTPUT:

Do's and Don'ts:
  • Do use rem and em for responsive designs.
  • Don't rely solely on px as it limits scalability.
  • Do specify a fallback font to ensure readability when custom fonts fail to load.
