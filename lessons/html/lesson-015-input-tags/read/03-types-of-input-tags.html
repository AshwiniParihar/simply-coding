Text Input

The text input allows users to enter a single line of plain text, such as names or general information.

Example: 
<input type="text" placeholder="Enter your name" />
Password Input

Password input masks characters to keep sensitive information secure while typing.

Example: 
<input type="password" placeholder="Enter your password" />
Email Input

Email input validates the entered text as a properly formatted email address.

Example: 
<input type="email" placeholder="Enter your email" />
Number Input

The number input restricts the field to only accept numerical values.

Example: 
<input type="number" min="0" max="100" placeholder="Enter a number" />
Date Input

The date input allows users to select a date from a built-in calendar.

Example: 
<input type="date" />
Checkbox Input

The <input type="checkbox"> element lets users select one or multiple options from a list. Each checkbox operates independently, meaning multiple checkboxes can be selected simultaneously.

  • Use Cases: Great for preferences, options where multiple selections are allowed, or yes/no choices.
  • Grouping Options: Use the same name attribute for related checkboxes to group them, especially when submitting form data.
  • Labeling: Always label each checkbox for accessibility. Use <label> to associate text with the checkbox.
Example: 
<input type="checkbox" id="option1" name="options" value="Option 1">
<label for="option1">Option 1</label>

<input type="checkbox" id="option2" name="options" value="Option 2">
<label for="option2">Option 2</label>

In the example above, the checkboxes allow the selection of both Option 1 and Option 2.

Best Practice: Always group and label checkboxes clearly to avoid confusion, especially in large forms.
Radio Button Input

The <input type="radio"> element allows users to select only one option from a predefined list. Radio buttons are mutually exclusive within a group, meaning only one can be selected at a time.

  • Use Cases: Ideal for single-choice selections, such as choosing a gender, payment method, or answer in a survey.
  • Grouping Options: To group radio buttons, give them the same name attribute. Only one radio button with the same name can be selected at a time.
  • Labeling: Use <label> tags to clearly indicate each choice for accessibility.
Example: 
<input type="radio" id="optionA" name="selection" value="A">
<label for="optionA">Option A</label>

<input type="radio" id="optionB" name="selection" value="B">
<label for="optionB">Option B</label>

In the example above, users can select either Option A or Option B, but not both, as they share the same name attribute.

Best Practice: Use descriptive labels and group related radio buttons under a <fieldset> with a <legend> to provide context for the selection.
Submit Button

The submit button submits the form data to the server for processing.

Example: 
<input type="submit" value="Submit" />
