Value Attribute

The value attribute sets an initial or default value for an input field. This value will appear in the input field when the page loads or resets.

Example: 
<input type="text" value="John Doe">

This example displays John Doe as the default text in the input field.

Readonly Attribute

The readonly attribute specifies that an input field is read-only. Users can view or copy the content, but they cannot modify it.

Example: 
<input type="text" value="John Doe" readonly>

This input displays John Doe as uneditable text.

Disabled Attribute

The disabled attribute disables the input field, making it unclickable. The value of a disabled input is not submitted with the form.

Example: 
<input type="text" value="John Doe" disabled>

This example makes John Doe unclickable and uneditable.

Size Attribute

The size attribute specifies the width of an input field in characters. This attribute is typically used to adjust the visible input width.

Example: 
<input type="text" size="30">

This sets the input field to display approximately 30 characters.

Maxlength Attribute

The maxlength attribute limits the number of characters a user can enter in the input field.

Example: 
<input type="text" maxlength="10">

This restricts the user to a maximum of 10 characters in the field.

Min & Max Attributes

The min and max attributes define a range of acceptable values for the input field.

Example: 
<input type="number" min="1" max="5">

This restricts the input to values between 1 and 5.

Pattern Attribute

The pattern attribute specifies a regular expression that an input field’s value must match for the form to be submitted. This attribute is useful for validating specific input formats, such as phone numbers, email addresses, or other patterns.

When using pattern, it’s recommended to include a title attribute to provide users with guidance on the required format, as browsers may not show an error message without it.

Example: 
<input type="text" pattern="[A-Za-z]{3}" title="Enter a three-letter country code">

In this example, the pattern [A-Za-z]{3} requires the user to enter exactly three alphabetical characters. If the input does not meet this criteria, the form will not submit, and the title text will appear as a hint.

Use Case: Validating country codes, postal codes, or other specific text formats without JavaScript.

Placeholder Attribute

The placeholder attribute provides a short hint that describes the expected input in an input field. This hint is displayed inside the input field as a faded or greyed-out text before the user enters a value.

Best Practices: The placeholder should be brief and should not be used as a substitute for labels. Use it to give users an idea of the expected data format, like “name@example.com” for an email input.

Example: 
<input type="text" placeholder="Enter your name">

Here, “Enter your name” will appear as faded text in the input field. Once the user starts typing, the placeholder text disappears.

Use Case: Providing additional guidance within the input field, especially for complex or less common data formats.

Required Attribute

The required attribute ensures that an input field is not left blank when a form is submitted. This attribute is commonly used in fields where critical information must be provided, such as usernames, passwords, and contact information.

When a required input field is left empty, the browser will prevent form submission and prompt the user to complete the field, typically displaying a browser-specific error message.

Example: 
<input type="text" required>

This input field must be filled out before the form can be submitted.

Use Case: Ensuring critical fields, such as “Email” or “Password,” are completed to avoid incomplete submissions.
