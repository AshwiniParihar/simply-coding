Validation of Input Fields

HTML provides built-in validation attributes that help ensure input fields meet specific requirements before form submission. Common attributes include required, pattern, min, and max.

Basic Validation Example

      <form action="submit.html" method="post">
  <fieldset>
    <legend>User Information</legend>
    <label for="name">Name:</label>
    <input type="text" id="name" required />
    <label for="age">Age:</label>
    <input type="number" id="age" min="18" max="100" required />
    <label for="email">Email:</label>
    <input type="email" id="email" pattern=".+@.+\.com" required />
  </fieldset>
  <button type="submit">Submit</button>
</form>
User Information

This form uses various validation attributes. The required attribute ensures the fields cannot be empty. The min and max attributes specify a range for numerical input, and the pattern attribute defines a regular expression to match email addresses ending in ".com".

Validation Attributes Explained

required

Ensures the input field is not left empty.

pattern

Defines a regex pattern that the input must match.

min

Sets the minimum value for numerical input fields.

max

Sets the maximum value for numerical input fields.

What is Regex?
  • Pattern Matching: Regex, short for "Regular Expressions," is used to define patterns in text, allowing you to search, match, or manipulate specific character sequences.
  • Flexible Searches: With regex, you can search for exact words, specific formats (like email addresses), or more complex patterns, such as all words starting with "a."
  • Validation: Regex is commonly used to validate input, such as checking if a phone number, email, or zip code is correctly formatted.
  • Text Replacement: Regex can be used to find and replace specific patterns in text, making it useful for reformatting or correcting strings.
  • Common Syntax: Key symbols include . for any character, * for zero or more repetitions, ^ for start of a line, and $ for end of a line.
required Attribute

The required attribute ensures the input field must be filled before submitting the form. It's suitable for fields like name, email, or phone number that need user input.

Example: 
<input type="text" required>
pattern Attribute

The pattern attribute uses a regular expression to define a required format for the input. It’s often applied to fields like email, phone numbers, or custom formats.

Example: 
<input type="email" pattern=".+@.+\.com">

This example accepts any email ending in ".com".

min Attribute

The min attribute sets a minimum acceptable value for numeric input. Useful for fields like age or quantity to limit entry to values above a specified threshold.

Example: 
<input type="number" min="18">

This example ensures the age entered is at least 18.

max Attribute

The max attribute sets a maximum allowable value for numeric input. It’s useful for restricting fields like age, quantity, or other numerical values.

Example: 
<input type="number" max="100">

This example restricts age input to a maximum of 100.
