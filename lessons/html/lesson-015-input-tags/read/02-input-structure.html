What is the <input> Tag?

The <input> tag is an essential part of HTML forms. It allows users to provide data to a website, enabling features like user login, search bars, and interactive forms. The <input> tag is self-closing, meaning it does not need a separate closing tag like </input>.

Basic Structure of Input Tags

      <input type="text" placeholder="Enter your text here">

The <input> tag has several key attributes, including type (which defines the kind of input, such as text, email, or password) and placeholder (which displays temporary text inside the field). Here’s a basic example of an input tag used for text input.

Common Types of Input Tags

Text

Accepts a single line of plain text.

Email

Ensures the input is in a valid email format.

Password

Masks the characters entered for security.

Number

Accepts numerical values only.

Date

Opens a calendar selector for date input.

Pro Tip: Choose the correct input type to improve usability and accessibility. For example, using type="email" provides a keyboard optimized for email entry on mobile devices.

Self-Closing Nature of Input Tags

The <input> tag is a self-closing tag, meaning it does not require an end tag like </input>. This is because it is an empty element with no content inside it, only attributes.

This structure keeps HTML code concise and is typical for elements that don’t contain other HTML elements.

Try It Out!

        
<input type="text" placeholder="Enter your name">
<input type="email" placeholder="Enter your email">
<input type="number" min="0" max="100" placeholder="Age">
Common Mistakes
  • Not using type="password" for sensitive information.
  • Leaving out the placeholder attribute, which helps guide users on what to enter.
  • Using the wrong input type, such as type="text" for email addresses.
  • Setting restrictive min and max values that may frustrate users.
Text Input

The text input type is one of the most commonly used fields, allowing users to enter single-line plain text.

Common Uses:
  • Names, titles, and basic identifiers.
  • Customizable fields for general text input.
Example: 
<input type="text" placeholder="Enter your name">
Notes:
  • Works well with maxlength and pattern attributes to restrict input format.
  • Consider using autocomplete="off" if you do not want the browser to store and suggest previous entries.
Email Input

The email input type ensures that any entered text follows the email format (e.g., user@example.com).

Common Uses:
  • Email collection for accounts, subscriptions, and contact forms.
Example: 
<input type="email" placeholder="you@example.com">
Notes:
  • Will trigger validation errors if the format is incorrect, even if it is left blank (if required).
  • Many browsers show a specific email keyboard on mobile devices, enhancing user experience.
Password Input

The password input type masks characters, displaying dots or asterisks to hide sensitive information.

Common Uses:
  • Input fields for account passwords, PINs, or sensitive codes.
Example: 
<input type="password" placeholder="Enter your password">
Notes:
  • Data entered in password fields is not stored in the browser's autofill history by default, enhancing privacy.
  • Can be paired with minlength and maxlength to set security standards for passwords.
Number Input

The number input type is tailored for numerical input and can be limited with min, max, and step attributes.

Common Uses:
  • Ages, quantities, prices, or other numeric data entries.
Example: 
<input type="number" min="0" max="100" placeholder="Enter your age">
Notes:
  • Browsers may provide up and down arrows for incremental input control, enhancing usability.
  • Pair with step to allow specific increments (e.g., step="5" for increments of 5).
Date Input

The date input type allows users to select a date from a date picker, providing a calendar view for ease.

Common Uses:
  • Date of birth, booking dates, appointments, or deadlines.
Example: 
<input type="date">
Notes:
  • The browser’s default calendar view is shown, ensuring format consistency.
  • Use min and max to set date ranges, preventing future/past dates as needed.
  • This input type provides a uniform experience across different devices, especially on mobile.
